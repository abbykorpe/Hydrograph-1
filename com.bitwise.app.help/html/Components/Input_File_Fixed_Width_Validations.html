Input File Fixed Width Validations

Accelero Release Version 1.0

Validations for the Input File Fixed Width component are in place for each property present on the Properties window.

General Properties:

  • File Path - File Path is a mandatory field and needs to be present. If absent, error for empty field appears along with a message on tool tip for user to fill it appropriately

  • Safe/Strict -These properties expect boolean values and have the default value as 'False' set for each one of them. In case user selects the option to parameterize these values, then it cannot remain blank and has validation error similar to the File Path.
  • Character Set -Character set has a default value of UTF-8. User may parameterize the character set similar to properties listed above. However, blank parameter values result in error validation and is not allowed.

  • Phase -Phase has a default value of 0. However, blank text box results in error validation and is not allowed.

Schema Types

  • Internal - User is provided a Grid to enter the internal schema of the Input Fixed Width file. Field name cannot remain blank.

  • External - User can provide an external Accelero Schema file of XML format. A text box is provided to either manually type or use the Browse button to select the schema file from File system. The External schema file path must not remain blank and has the validation error message if user does not fill the text box
