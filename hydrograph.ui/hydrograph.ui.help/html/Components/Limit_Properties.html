Limit Properties

Properties for the Limit component can be viewed by Double click->component on canvas. The properties contain only the 'General' tab since Limit falls under the Straight Pull category in the component palette.

General Properties

Display

  • Name - The identifier for the component. This is a mandatory property. This property is pre-populated with the component name, i.e. 'Limit' followed by an incremental number. It can be changed to any custom name. The name property has following restrictions:
    • Must be specified and should not be blank.
    • Must be unique across the job.
    • Accepts only alphabets (a-z), numerals (0-9) and 4 special characters: "_", "-", ",", " " (space).
  • Base Type - Base Type signifies the category of the component. Limit is of base type Straight Pull. This is a non editable field.
  • Type - Type defines the type of component within the category. This typically is the name of the component. This is a non editable field.

Configuration

  • Count - Count accepts an integer value that specifies the number of input records user wishes to transmit to the next component.
  • Runtime Properties - Runtime properties are used to override the Hadoop configurations specific to this Limit component at run time. User is required to enter the Property Name and Property Value in the runtime properties grid.

    Check How to pass Hadoop properties to component

  • Phase - Phase accepts an integer value and signifies the phase this component will execute in. The default value for phase is 0. Phase can have a maximum value of 99. Phase is a mandatory property.
