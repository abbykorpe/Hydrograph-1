Aggregate Properties

Properties for the Aggregate component can be viewed by Double click->component on canvas. The properties contain a 'General' tab and a 'Schema' tab. Common and mandatory properties are present in the General tab. Schema displays option to accept the field schema i.e. field name, Data type, scale etc.

General Properties:

Display

  • Name - The Name field will specify the name of the component as desired by the user
  • Base Type - Base Type signifies the basic operation area of the component. In this case it is of Base Type Transform.
  • Type - Type further categorizes the component to detailed operation that it intends to perform. Here it is of type Aggregate.

Configuration

  • Runtime Properties - Runtime properties are used to override the Hadoop configurations specific to Aggregate component at run time. User is required to enter the Property Name and Value in the runtime properties grid.

  • Key Field - Key Field opens up a grid to accept the key fields that will be used to group the data fields for performing aggregation operations

  • Secondary Keys - Secondary keys opens up a grid that accepts the columns that will be used for ordering the output of Aggregation. Here, user can specify the Sort Order which can be either Ascending or Descending. By default the sort order is Ascending(Asc).

  • Aggregate - Edit option opens up the 'Operation Editor' view, where the user can select the input fields needed for the aggregation operations, write custom operation classes (or select already existing standard operation classes) to use these input fields and then extract the output fields all in the same grid view. Click on Operation Class provided in the Operation Class section of the Help content to know more about creating and using Standard classes here.

  • Phase - Phase simply accepts a numeric number starting from 0 and signifies the phase this component will execute in. By default this is 0.

Schema tab:

User can define record structure of Aggregate component in this tab by specifying below attributes.

  • Field Name - User needs to enter all the field names in this column
  • Data type - By default "String" data type has been selected for any field added to the Grid. User can select from different data type values like "Integer", "Double", "Float" etc.
  • Scale - Scale for a Double, Float or BigDecimal field can be specified here.
  • Scale Type – Scale Type accepts values as implicit or explicit for BigDecimal field and none for other data types. Explicit considers the length of '.' and implicit ignores length of '.' for the BigDecimal field.
  • Date Format - User can specify desired date format. Refer to java date formats page for acceptable date formats.
  • Precision – User can specify precision for the field.
  • Field Description – User can specify description for the field.

Pull Schema – User can click on Pull Schema and the fields defined in Operation Editor's output, will be pulled here.
