Input Hive Parquet Validations

Accelero Release Version 1.0

Validations for the Input Hive Parquet component are in place for each property present on the Properties window.

General Properties:

  • Database Name - Database Name is a mandatory field and needs to be present. If absent, error for empty field appears along with a message on tool tip for user to fill it appropriately

  • Table Name - Table Name is a mandatory field and needs to be present. If absent, error for empty field appears along with a message on tool tip for user to fill it appropriately

  • External Table Path -External Table Path is an optional field. Enter null if not required.

  • Partition Keys -Partition Keys is an optional field. If absent, it does not create any partitions on the data.

  • Phase -Phase has a default value of 0. However, blank text box results in error validation and is not allowed.

Schema Types

  • Internal - User is provided a Grid to enter the internal schema of the Input Parquet file. Field name cannot remain blank.

  • External - User can provide an external Accelero Schema file of XML format. A text box is provided to either manually type or use the Browse button to select the schema file from File system. The External schema file path must not remain blank and has the validation error message if user does not fill the text box
